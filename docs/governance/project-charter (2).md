# Project Charter
## Last Updated: 2026-07-05

---

## 1. Overview
The Project Charter defines the purpose, scope, objectives, constraints, and governance of the Hybrid Homelab Project.  
It establishes the foundational intent and ensures all future work aligns with the project’s mission and architectural direction.

---

## 2. Scope
This charter applies to:
- Identity services (AD DS, PKI, ADFS, Azure AD Connect)
- Network infrastructure (Cisco L3, VLANs, ACLs)
- Compute infrastructure (Hyper‑V, ESXi)
- Monitoring, backup, and disaster recovery
- Documentation, governance, and workflow standards

---

## 3. Objectives
- Build an enterprise‑grade hybrid identity and infrastructure environment.
- Practice real‑world architecture, design, and operational workflows.
- Maintain strict documentation and Git discipline.
- Simulate professional IT project execution end‑to‑end.

---

## 4. Constraints
- Homelab hardware limitations  
- Time availability  
- Single‑engineer execution  
- Strict adherence to Master Document + Kanban (Holy Grails)

---

## 5. Deliverables
- Architecture documents  
- Design documents  
- Deployment procedures  
- Rollback procedures  
- Validation checklists  
- ADRs  
- Monitoring & DR plans  
- Fully documented Git workflow

---

## 6. Acceptance Criteria
- Charter is approved and stored under `docs/governance/`
- All future tasks reference this charter when defining scope
- Charter remains updated as the project evolves
